AgentSmark Frontend Application Documentation

Comprehensive technical manual for the React + TypeScript frontend dashboard client

Application Overview

The AgentSmark Frontend is an interactive single-page application client built with React, TypeScript, and the Vite build toolchain. It acts as a user dashboard allowing users to:

  • Submit natural language searches to find stream feeds for Movies, TV Shows, and Anime via LLM-driven query resolution.
  • Browse real-time listings of live sports events currently broadcasting or scheduled to start, including multi-source link selectors.
  • Monitor analytics and stream availability seamlessly.

To ensure a responsive user experience reminiscent of the classic web era, the client uses a custom visual style system written in optimized vanilla CSS that emulates the classic Windows XP desktop and browser chrome, retro beveled input fields, and digital counters. Telemetry and analytics are integrated using Vercel Analytics.

src/App.tsx Root Component

The root layout component of the React application. It manages the active tab state, wraps the entire UI inside a Windows XP window frame mimicking Microsoft Internet Explorer (including a title bar with minimize/maximize/close controls, a mock menu bar, and an interactive address bar), and renders retro components like a scrolling marquee banner, a digital visit hit counter, and 88x31 web badges.

src/components/QueryPopup.tsx Query Component

Implements the search interface, styled in a vintage Yahoo! 2000 directory layout. Allows users to search for Sports, Movies, and TV streams. Directly below the search box, the component renders two independent, auto-scrolling horizontal carousels:

  • Trending Movies & TV Shows: Displays trending movie and TV show cards.
  • Live & Upcoming Sports: Displays active live and upcoming sports stream cards. Each sports card features a retro gradient background displaying the matchup's team initials, floating sport-specific emojis, and a "🔴 LIVE" status badge.

Both rows support independent horizontal auto-scrolling that pauses on hover (using separate hover states and refs), alongside manual navigation buttons. In strict adherence to AGENTS.md Rule 5, the sports carousel renders a clean empty state feedback message (No active sports streams currently broadcasting) when no active sports streams are retrieved from the backend.

src/components/LiveSports.tsx Sports Component

Renders active live sports listings. Executes a GET request to the backend /streams endpoint on mount. Groups fixtures, displays "LIVE" pulses or "UPCOMING" start time tags, provides search filters for filtering teams (such as Lakers or Chelsea) and categories via a sport category scroller (All Sports, Basketball, Football/Soccer, Soccer, NFL, Baseball), and displays multiple action buttons linking to alternative stream feeds. The filter logic is configured so that selecting either "Soccer" or "Football" displays both soccer and football matches, while "NFL" displays "american-football" fixtures.

src/components/Tables.tsx Tables Component

Implements the "The Tables" gaming and streaming view. Features register/login forms connecting to FastAPI auth routes, a lobby showing active sports categories (Basketball, Football/Soccer, Soccer, NFL, Baseball) and active streaming fixtures, a Game Station split-screen view with embed iframe player and player props (Over/Under picks) loaded directly in the "Wanna Bet?" panel, a player props search/filtering bar to search by player, team, or prop, a parlay slip builder for placing 2-5 leg parlays, and active parlay trackers with real-time simulated progress feeds and floating alert cards that update when props increase or settle. Now features a multi-channel stream/channel selector tab system in the Game Station to choose from multiple live broadcast links, active durations calculated dynamically (120 minutes for Soccer/Football, 180 minutes for NFL/American Football), customized tab category filters showing soccer/football matches on both "Soccer" and "Football" tabs while "NFL" displays "american-football" fixtures, and highlights pitching strikeouts / strikeouts props with a premium gold/amber theme (distinct badge, border, and glow effects) to make them prominent in the props selection tab, parlay slip, and placed parlays history list.

src/components/N64SportsIcons.tsx Retro Assets Component

Implements a zero-dependency, self-contained SVG graphic and team badge asset system. Houses the N64 visual filter definitions (scanlines, drop shadows, 3D metallic gold bevels) and includes a comprehensive team logo and category icon registry for NFL (32 teams), NBA (30 teams), MLB (30 teams), Soccer (top global clubs), F1 Constructors, and Combat Sports (UFC, WWE, AEW). Provides matching/normalization helpers to dynamically map API names to custom vintage console-styled polygonal and pixelated SVGs.

src/components/VisitorCounter.tsx Visitor Counter Component

Renders a classic Windows Solitaire-themed visitor counter component on a green felt background. Displays each digit of the visitor count as a realistic playing card with top-left and bottom-right suit indicators and color-coded digit values (red and black). Highlights the jackpot milestone at visitor #1000 with a custom green-felt solitaire jackpot banner that prompts the user to claim their prize.

src/components/PrizeModal.tsx Prize Modal Component

Renders a retro Windows Solitaire Grand Prize Modal featuring a classic beveled Windows 95/98 dialog box design. Styled with a deep green felt interior, a visual fan of cards representing a Royal Flush of Spades, card suit indicators, and a classic beveled Windows 95 claim button that credits the user with SFO Bucks.

Project Configuration

1. Vite Build System

Configured via vite.config.ts. Leverages Vite's hot module replacement (HMR) and fast bundlers. Runs the @vitejs/plugin-react plugin for React runtime compilation.

2. Environment Variables

The client interacts with the backend microservice using the following configuration environment variable:

  • VITE_API_BASE_URL: Specifies the root URL of the running FastAPI backend. If this variable is omitted, the app defaults to https://agent-smark-backend.onrender.com.

3. Tailwind CSS Integration

The project is fully compatible with Tailwind CSS integration using standard Vite plugins. However, the current release uses custom vanilla styling (found in src/App.css and src/styles/) to maintain zero build-time styling overhead, maximize rendering speed, and eliminate additional postcss configuration complexity.

4. Vercel Analytics

Imported and executed inside App.tsx using the Vercel Analytics SDK (@vercel/analytics). The call to inject() runs automatically at launch, collecting page views, event tracking, and performance telemetry when hosted on the Vercel platform.

AgentSmark Frontend Application Documentation

Comprehensive technical manual for the React + TypeScript frontend dashboard client

Application Overview

The AgentSmark Frontend is an interactive single-page application client built with React, TypeScript, and the Vite build toolchain. It acts as a user dashboard allowing users to:

  • Submit natural language searches to find stream feeds for Movies, TV Shows, and Anime via LLM-driven query resolution.
  • Browse real-time listings of live sports events currently broadcasting or scheduled to start, including multi-source link selectors.
  • Monitor analytics and stream availability seamlessly.

To ensure a premium responsive user experience without heavy package overhead, the client uses a custom visual style system written in optimized vanilla CSS. Telemetry and analytics are integrated using Vercel Analytics.

src/App.tsx Root Component

The root layout component of the React application. It injects and triggers Vercel Analytics, manages the active tab state (swapping between the search interface and live sports listing), renders a responsive header logo, tab navigation bar, hero description panel, and structural footer.

src/components/QueryPopup.tsx Query Component

Implements the search overlay interface. Allows users to choose a category (Sport, Movie, TV), input queries, and POST requests to the backend /watch endpoint. Contains result parsers to render movie metadata, poster cards fetched from TMDB, sports vs. match badges, dynamic initials/colors for team fallbacks, and redirects to external streams.

src/components/LiveSports.tsx Sports Component

Renders active live sports listings. Executes a GET request to the backend /streams endpoint on mount. Groups fixtures, displays "LIVE" pulses or "UPCOMING" start time tags, provides search filters for filtering teams (such as Lakers or Chelsea), and displays multiple action buttons linking to alternative stream feeds.

Styling & Sheets Layout

Rather than introducing massive component library dependencies or preprocessors, the frontend utilizes optimized custom vanilla CSS. Each style sheet is mapped directly to components and imported locally:

  • src/App.css: Configures global page typography (Plus Jakarta Sans font loading), header glassmorphism effects (using backdrop-filter and border outlines), flex alignments, navigation buttons active/hover states, and general responsive spacing constraints.
  • src/styles/QueryPopup.css: Styles the query form layout, category chips, interactive inputs, loaders, and search result grids. Arranges TMDB movie cards with poster layouts, gradients, overlay titles, and responsive sports cards.
    *Note: Although referenced under src/components/styles/ in legacy guides, these sheets reside inside src/styles/ in the current codebase layout.
  • src/styles/LiveSports.css: Dictates the layout of sports schedules. Manages upcoming/live badge backgrounds, live status indicator animations (pulsing dot keyframe), team-vs grid divisions, and channel stream buttons.
  • src/styles/VideoPlayerModal.css: Cleared / Deprecated. Empty styling sheet associated with the deprecated video player modal.
  • src/styles/Tables.css: Layout and presentation rules for the Tables component. Configures the split-screen workspace grids, auth card formatting, prop selection buttons active states, parlay slip layouts, and keyframe animations for floating update alert cards.

Project Configuration

1. Vite Build System

Configured via vite.config.ts. Leverages Vite's hot module replacement (HMR) and fast bundlers. Runs the @vitejs/plugin-react plugin for React runtime compilation.

2. Environment Variables

The client interacts with the backend microservice using the following configuration environment variable:

  • VITE_API_BASE_URL: Specifies the root URL of the running FastAPI backend. If this variable is omitted, the app defaults to https://agent-smark-backend.onrender.com.

3. Tailwind CSS Integration

The project is fully compatible with Tailwind CSS integration using standard Vite plugins. However, the current release uses custom vanilla styling (found in src/App.css and src/styles/) to maintain zero build-time styling overhead, maximize rendering speed, and eliminate additional postcss configuration complexity.

4. Vercel Analytics

Imported and executed inside App.tsx using the Vercel Analytics SDK (@vercel/analytics). The call to inject() runs automatically at launch, collecting page views, event tracking, and performance telemetry when hosted on the Vercel platform.

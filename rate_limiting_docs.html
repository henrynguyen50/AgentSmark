Gemini Search Endpoint Rate Limiting

This documentation details the implementation of rate limiting on the Gemini search endpoint (/watch) in the backend of the AgentSmark project.

Overview

To prevent abuse, quota exhaustion, and unnecessary API costs, the /watch endpoint is protected by a rate limiter. It restricts clients to a configurable limit (defaulting to 5 requests per minute) per IP address.

Key Technologies Used

  • FastAPI: The web framework used to build the backend API.
  • SlowAPI: A rate-limiting library for FastAPI built on top of limits.

Implementation Details

The rate limiting logic is configured inside agent-smark-backend/main.py:

1. Proxy-Aware IP Resolution

To support deployments behind reverse proxies (such as Cloudflare, Vercel, or AWS Load Balancers), the limiter uses a custom get_ip_address function rather than the default remote address helper. It parses the client IP address from the standard X-Forwarded-For header:

def get_ip_address(request: Request) -> str:
    forwarded_for = request.headers.get("x-forwarded-for")
    if forwarded_for:
        return forwarded_for.split(",")[0].strip()
    return request.client.host if request.client else "127.0.0.1"

For details on how the X-Forwarded-For header works, refer to the MDN Web Docs.

2. Endpoint Decoration

The /watch endpoint is annotated with the limiter decorator:

@app.post("/watch")
@limiter.limit(RATE_LIMIT_WATCH)
def watch(request: Request, req: QueryRequest):
    ...

3. Exception Handling

When a client exceeds the defined threshold, SlowAPI raises a RateLimitExceeded exception. This is handled globally using _rate_limit_exceeded_handler, returning an HTTP status code 429 Too Many Requests with a detailed JSON payload.

Configuration

The rate limit is controlled by the environment variable RATE_LIMIT_WATCH defined in the .env file. If the environment variable is missing or empty, it gracefully falls back to the default:

  • Environment Variable: RATE_LIMIT_WATCH
  • Default Limit: 5/minute (5 requests per minute)
  • Example Formats: 10/minute, 200/day, 5/second
Note on Serverless Deployments

SlowAPI tracks rate limits in-memory by default. If the backend is scaled horizontally across multiple servers or instances, a distributed storage backend like Redis should be integrated by configuring the storage_uri parameter on the Limiter instance.

Unit & Integration Tests

We verify rate limiting using mock-based integration tests. The test file is located at agent-smark-backend/test_rate_limiter.py.

To run the test suite, navigate to the backend directory and execute:

python -m unittest test_rate_limiter.py

The tests cover:

  • Basic Rate Limiting: Confirms 5 consecutive requests return status 200 and the 6th request returns status 429.
  • IP Isolation: Confirms that requests originating from different X-Forwarded-For headers are tracked in separate limit buckets.
